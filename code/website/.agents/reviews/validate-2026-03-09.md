## Issues Found

### Critical (fix before commit)

#### 1. `.env` file tracked in git history
- **File**: `.env` (shown as modified in `git status`)
- **Issue**: Environment file with Supabase credentials appears in git tracking
- **Why**: Secrets exposed in version control can be extracted from git history even after removal
- **Fix**: Add `.env` to `.gitignore`, remove from tracking with `git rm --cached .env`, rotate all exposed credentials (Supabase anon key, any service keys)

#### 2. JSON-LD `</script>` injection in OG endpoint
- **File**: `api/og/[id].ts:112`
- **Issue**: Car data (omschrijving, merk, model) is inserted into `<script type="application/ld+json">` without sanitizing `</script>` sequences
- **Why**: A car description containing `</script><script>alert(1)</script>` would break out of the JSON-LD block and execute arbitrary JavaScript
- **Fix**: Add a `sanitizeJsonLd()` helper that replaces `</` with `<\/` in the serialized JSON string before embedding in HTML:
  ```typescript
  const safeJsonLd = cleanJsonLd.replace(/<\//g, '<\\/');
  ```

#### 3. XML injection in sitemap
- **File**: `api/sitemap.ts:50`
- **Issue**: Car IDs are interpolated directly into XML `<loc>` tags without escaping
- **Why**: If a car ID contains `&`, `<`, or `>` characters, the XML becomes malformed and unparseable by search engines
- **Fix**: Add an `escapeXml()` helper:
  ```typescript
  function escapeXml(str: string): string {
    return str.replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;');
  }
  ```

#### 4. No UUID validation on OG route parameter
- **File**: `api/og/[id].ts:8-11`
- **Issue**: The `id` parameter is used directly in a Supabase query without UUID format validation
- **Why**: Malformed IDs could cause unexpected database behavior; UUID validation acts as a first defense layer
- **Fix**: Add UUID regex check before querying:
  ```typescript
  const UUID_RE = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;
  if (!UUID_RE.test(id)) return res.redirect(302, SITE_URL);
  ```

#### 5. 10 high-severity dependency vulnerabilities
- **File**: `package.json` / `node_modules`
- **Issue**: `npm audit` reports 10 high-severity CVEs including react-router-dom XSS and rollup vulnerabilities
- **Why**: Known vulnerabilities in production dependencies can be exploited
- **Fix**: Run `npm audit fix` and update affected packages. If breaking changes, pin to patched minor versions.

### Important (fix before PR)

#### 6. Missing try-catch in all 3 API routes
- **File**: `api/llms.ts`, `api/sitemap.ts`, `api/og/[id].ts`
- **Issue**: No try-catch blocks around Supabase queries or response generation. If Supabase throws, the serverless function crashes with a 500 and no useful error info.
- **Why**: Unhandled exceptions in Vercel serverless functions return opaque 500 errors to users and bots
- **Fix**: Wrap each handler body in try-catch, return a meaningful error response:
  ```typescript
  try {
    // ... existing handler logic
  } catch (err) {
    console.error('API error:', err);
    return res.status(500).send('Internal Server Error');
  }
  ```

#### 7. Supabase error fields ignored in all 3 API routes
- **File**: `api/llms.ts:17,25`, `api/sitemap.ts:23`, `api/og/[id].ts:22`
- **Issue**: Supabase queries destructure `{ data }` but never check `{ error }`. Failed queries silently return empty data.
- **Why**: Database errors (RLS, network, schema changes) are completely invisible — the endpoint returns incomplete/empty data instead of failing visibly
- **Fix**: Destructure and check error:
  ```typescript
  const { data, error } = await supabase.from('cars').select(...);
  if (error) {
    console.error('Supabase query error:', error);
    return res.status(500).send('Database error');
  }
  ```

#### 8. Missing Content-Security-Policy header
- **File**: `vercel.json:90-101`
- **Issue**: Security headers include X-Frame-Options, HSTS, etc. but no CSP header
- **Why**: CSP is a critical defense layer against XSS, clickjacking, and data injection attacks
- **Fix**: Add a CSP header to the global headers block:
  ```json
  { "key": "Content-Security-Policy", "value": "default-src 'self'; script-src 'self' 'unsafe-inline' https://va.vercel-scripts.com; style-src 'self' 'unsafe-inline'; img-src 'self' https://*.supabase.co data:; connect-src 'self' https://*.supabase.co; font-src 'self'" }
  ```

#### 9. `siteSettingsService.ts` bypasses TypeScript safety
- **File**: `src/services/siteSettingsService.ts`
- **Issue**: Uses `(supabase as any)` casts to bypass type checking, no try-catch, no logging
- **Why**: TypeScript safety exists to catch bugs at compile time. `as any` defeats this purpose and hides real type mismatches.
- **Fix**: Use proper Supabase generated types. If the table isn't in the generated types yet, regenerate types with `supabase gen types typescript`.

#### 10. `getSimilarCars` silently swallows query errors
- **File**: `src/services/carService.ts` — `getSimilarCars` function
- **Issue**: The second Supabase query (fallback query for similar cars) catches errors silently and returns an empty array
- **Why**: If the fallback query fails due to a real issue (schema change, RLS), the user sees "no similar cars" instead of an error, making debugging very hard
- **Fix**: Log the error even if returning empty array:
  ```typescript
  catch (err) {
    console.error('getSimilarCars fallback error:', err);
    return [];
  }
  ```

#### 11. `hexToHSL` and `applyColors` silent null returns
- **File**: `src/hooks/useSiteSettings.ts`
- **Issue**: `hexToHSL` returns null on invalid hex without logging. `applyColors` silently skips when colors are invalid.
- **Why**: If site settings contain a malformed color value, the entire theme silently breaks with no indication of what went wrong
- **Fix**: Add console.warn for invalid inputs so developers can diagnose theme issues
