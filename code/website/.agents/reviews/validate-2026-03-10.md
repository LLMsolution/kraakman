## Issues Found

### Critical (fix before commit)

**1. `siteUrl` still points to wrong domain in Edge Function**
- **File**: `supabase/functions/send-testdrive-request/index.ts:22`
- **Issue**: `siteUrl: "https://autoservicevanderwaals.nl"` — car links in confirmation emails point to wrong domain
- **Why**: Customers receive emails with broken "Bekijk op website" links
- **Fix**: Change to `"https://wkautoselectie.nl"` (or the actual live domain)

**2. `siteUrl` and email in business config still wrong domain**
- **File**: `src/config/business.ts:5,9`
- **Issue**: `SITE_URL: "https://autoservicevanderwaals.nl"` and `EMAIL: "info@autoservicevanderwaals.nl"`
- **Why**: Used as fallback throughout the app (SEO, FAQ schema, etc.)
- **Fix**: Update to `wkautoselectie.nl` domain

**3. API endpoints use wrong domain**
- **File**: `api/sitemap.ts:4`, `api/og.ts:4`, `api/llms.ts:4,105`
- **Issue**: All use `"https://autoservicevanderwaals.nl"` as `SITE_URL`
- **Why**: Sitemap, OG meta tags, and LLM context point to wrong domain
- **Fix**: Update all to correct domain

**4. Content defaults use wrong email**
- **File**: `src/data/contentDefaults.ts:102`
- **Issue**: `email: "info@autoservicevanderwaals.nl"`
- **Why**: Default contact email shown to users when site settings fail to load
- **Fix**: Change to `"info@wkautoselectie.nl"`

### Important (fix before PR)

**5. `getBrand()` has empty catch block in all Edge Functions**
- **File**: `supabase/functions/send-contact/index.ts:58`, `send-feedback/index.ts:55`, `send-testdrive-request/index.ts:58`
- **Issue**: `catch (_) { /* fallback to defaults */ }` — silently swallows all database errors
- **Why**: If site_settings table is broken, emails use hardcoded defaults with no logging. Debugging is impossible.
- **Fix**: Add `console.error('getBrand() failed:', _);` before the fallback

**6. Customer confirmation email failure returns success**
- **File**: `supabase/functions/send-contact/index.ts:230-232`, `send-testdrive-request/index.ts:278-280`
- **Issue**: When confirmation email fails, only `console.error` is called but `{ success: true }` is returned
- **Why**: Customer doesn't receive confirmation, but the response says success. Business email already sent, so partial failure.
- **Fix**: Consider returning `{ success: true, warning: "Bevestigingsmail kon niet worden verzonden" }` so the frontend can display a partial success message

**7. `send-feedback` lacks Turnstile verification**
- **File**: `supabase/functions/send-feedback/index.ts`
- **Issue**: Unlike send-contact and send-testdrive-request, this function has no Turnstile captcha check
- **Why**: Spam bots can abuse the feedback form
- **Fix**: Add the same `verifyTurnstile` pattern from the other two functions

**8. `carImage` URL not validated before injection into email HTML**
- **File**: `supabase/functions/send-testdrive-request/index.ts:174-178`
- **Issue**: `carImage` is sanitized with `sanitize()` (HTML entities) but no URL validation. A `javascript:` or data URI could pass.
- **Why**: While email clients don't execute JS in img src, it's defense-in-depth
- **Fix**: Validate that `carImage` starts with `https://` before using it

**9. No rate limiting on Edge Functions**
- **File**: All 3 Edge Functions
- **Issue**: No rate limiting — an attacker can send unlimited emails
- **Why**: Could exhaust Resend API quota, cause billing issues, or be used for email bombing
- **Fix**: Implement rate limiting at Cloudflare/Supabase level, or add simple IP-based throttling

**10. CORS allows all origins**
- **File**: All 3 Edge Functions (`"Access-Control-Allow-Origin": "*"`)
- **Issue**: Any website can call your Edge Functions
- **Why**: Combined with lack of rate limiting, this is exploitable
- **Fix**: Restrict to `https://wkautoselectie.nl` and `https://autoservicevanderwaals.nl`
